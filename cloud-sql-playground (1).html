การฝึก SQL แบบเดิม เสียเวลาไปกับสิ่งที่ไม่ใช่การเรียน

ก่อนจะได้เขียน query แรก นักศึกษาต้องผ่านด่านของการติดตั้งและตั้งค่าเครื่องมือ ซึ่งไม่เกี่ยวอะไรกับเนื้อหาวิชาเลย

ERROR

ต้องติดตั้ง PostgreSQL / MySQL เอง ใช้เวลานานและมักติดปัญหาทางเทคนิค

ERROR

เวอร์ชันซอฟต์แวร์ต่างกันในแต่ละเครื่อง ทำให้ผลลัพธ์การทำงานไม่ตรงกัน

ERROR

การนำเข้า dataset ตัวอย่างยุ่งยากสำหรับผู้เริ่มต้น

ERROR

เมื่อเครื่องมีปัญหาหรือระบบล่ม ต้องติดตั้งใหม่ทั้งหมด กระทบความต่อเนื่องในการเรียน

ERROR

อาจารย์เสียเวลาเตรียม environment ให้นักศึกษาซ้ำ ๆ ทุกภาคการศึกษา และตรวจงานด้วยมือ

แนวทางแก้ไข

เปิดเบราว์เซอร์ Login แล้วเริ่มเขียน SQL ได้ทันที

Cloud SQL Playground ย้ายทุกอย่างขึ้น cloud ให้นักศึกษาสร้างฐานข้อมูลส่วนตัว นำเข้าข้อมูล และฝึกเขียน query ได้จากอุปกรณ์ใดก็ได้ที่ต่ออินเทอร์เน็ต

01

Login

ยืนยันตัวตนผ่าน Amazon Cognito พร้อม role-based access control

02

SQL Editor

เขียนและรันคำสั่งผ่านเบราว์เซอร์ พร้อม syntax highlighting

03

Upload dataset

นำเข้าไฟล์ CSV หรือ Excel เข้า schema ของตัวเองได้ทันที

04

Execute SQL

รันคำสั่งจริงบน Aurora Serverless พร้อมแสดงผลแบบตาราง

05

Query history

ย้อนดูและรัน query เก่าซ้ำได้ พร้อม export เป็น CSV/JSON

06

Auto-delete

ลบ playground อัตโนมัติเมื่อไม่ใช้งานเกิน 30 นาที ประหยัดทรัพยากร

07

Assignment

อาจารย์สร้างโจทย์ นักศึกษาส่ง SQL ระบบตรวจให้คะแนนอัตโนมัติ

08

Sample library

เลือกใช้ dataset ตัวอย่าง เช่น Northwind, Sakila ได้ทันทีโดยไม่ต้อง upload เอง

Use case specification

11 use case ครอบคลุมทั้งสามบทบาท

คลิกเพื่อดู basic flow ของแต่ละ use case

UC-01LoginStudent · Instructor · Admin›

ผู้ใช้เข้าสู่ระบบเพื่อยืนยันตัวตนและรับสิทธิ์การใช้งานตามบทบาทของตน

Basic flow
  1. เปิดหน้าเว็บและกรอก email/password
  2. ระบบส่งคำขอยืนยันตัวตนไปยัง Amazon Cognito
  3. Cognito ตรวจสอบและส่ง token กลับ (JWT)
  4. ระบบตรวจสอบ role จาก token แล้วนำไปหน้า dashboard ตามบทบาท
UC-02Create playgroundStudent›

นักศึกษาสร้างฐานข้อมูลส่วนตัว (playground) เพื่อใช้ฝึกปฏิบัติ SQL

Basic flow
  1. กด "Create playground" จากหน้า dashboard
  2. ระบบตรวจสอบจำนวน playground เทียบกับ resource quota
  3. เรียก Playground Provisioning Lambda
  4. Lambda สร้าง schema ใหม่บน Aurora พร้อม database user สิทธิ์จำกัด
  5. บันทึก metadata ลง DynamoDB แล้วเปิดหน้า SQL editor
UC-03Upload datasetStudent · Instructor›

ผู้ใช้อัปโหลดไฟล์ CSV หรือ Excel เพื่อนำเข้าข้อมูลสู่ playground หรือ assignment

Basic flow
  1. เลือกไฟล์ CSV หรือ Excel จากเครื่อง
  2. ระบบตรวจสอบขนาดไฟล์เทียบกับ quota แล้วอัปโหลดไปยัง S3
  3. S3 trigger เรียก Dataset Import Lambda
  4. Lambda อ่านไฟล์ แปลงเป็นตาราง และ insert เข้า schema ของผู้ใช้
UC-04Execute SQL queryStudent›

ผู้ใช้เขียนและรันคำสั่ง SQL ผ่าน SQL editor เพื่อดูผลลัพธ์

Basic flow
  1. พิมพ์คำสั่ง SQL แล้วกด "Run"
  2. Query Execution Lambda ตรวจสอบตาม whitelist ที่อนุญาต
  3. รันคำสั่งด้วย database user สิทธิ์จำกัด พร้อม timeout 10 วินาที
  4. จำกัดผลลัพธ์ไม่เกิน 1,000 แถว แล้วอัปเดต last_active_at
UC-05Save & view query historyStudent›

ผู้ใช้บันทึกคำสั่ง SQL ที่เคยรัน และย้อนดูหรือรันซ้ำได้

Basic flow
  1. ระบบบันทึกคำสั่ง เวลา และผลลัพธ์โดยย่อลง DynamoDB หลังรันสำเร็จ
  2. เปิดเมนู "Query history" เพื่อดูรายการย้อนหลัง
  3. เลือก query แล้วกด "Run again" เพื่อรันซ้ำ
UC-06Export query resultStudent›

ผู้ใช้ส่งออกผลลัพธ์ของ query เป็นไฟล์ CSV หรือ JSON

Basic flow
  1. กด "Export" บนหน้าผลลัพธ์
  2. เลือกรูปแบบไฟล์ที่ต้องการ
  3. ระบบแปลงผลลัพธ์และส่งให้เบราว์เซอร์ดาวน์โหลด
UC-07Create assignmentInstructor›

อาจารย์สร้างโจทย์แบบฝึกหัด พร้อมกำหนด dataset ตั้งต้นและเฉลยสำหรับตรวจอัตโนมัติ

Basic flow
  1. กรอกชื่อโจทย์ คำอธิบาย และกำหนด due date
  2. อัปโหลด dataset ตั้งต้น
  3. กำหนดคำสั่ง SQL เฉลยสำหรับใช้ตรวจอัตโนมัติ
  4. บันทึก assignment แล้วแสดงในรายการให้นักศึกษาเห็น
UC-08Submit & auto-grade assignmentStudent · System›

นักศึกษาส่งคำสั่ง SQL เพื่อทำโจทย์ และระบบตรวจให้คะแนนอัตโนมัติ

Basic flow
  1. เขียนคำสั่ง SQL ตามโจทย์แล้วกด "Submit"
  2. ระบบส่ง event ไปยัง Grading Lambda ผ่าน EventBridge/SQS
  3. Lambda รันคำสั่งใน sandbox schema แยกจากข้อมูลจริง
  4. เปรียบเทียบผลลัพธ์กับเฉลยแบบ row-by-row แล้วคำนวณคะแนน
  5. บันทึกคะแนนและแสดงผลแบบ real-time
UC-09Auto delete idle playgroundSystem›

ระบบลบ playground ที่ไม่มีการใช้งานเกินระยะเวลาที่กำหนด เพื่อคืนทรัพยากร

Basic flow
  1. CloudWatch Events trigger Auto-Cleanup Lambda ทุก 5 นาที
  2. Lambda ดึง last_active_at ของทุก playground จาก DynamoDB
  3. ไม่ใช้งานครบ 25 นาที → แจ้งเตือนล่วงหน้า 5 นาที
  4. ไม่ใช้งานครบ 30 นาที → ลบ schema, ไฟล์ dataset และ metadata
UC-10Manage usersAdmin›

Admin จัดการบัญชีผู้ใช้ในระบบ — สร้าง แก้ไข ระงับ หรือลบ

Basic flow
  1. เปิดหน้า "User management" เพื่อดูรายการผู้ใช้ทั้งหมด
  2. เลือกดำเนินการ: เพิ่มผู้ใช้ / แก้ไข role / ระงับ / ลบบัญชี
  3. ระบบส่งคำขอไปยัง Amazon Cognito เพื่ออัปเดตข้อมูล
UC-11Monitor systemAdmin›

Admin ติดตามสถานะการทำงานของระบบ เช่น จำนวน playground ที่ active และการใช้ทรัพยากร

Basic flow
  1. เปิดหน้า "System dashboard"
  2. ระบบดึงข้อมูลจาก CloudWatch (logs, metrics) และ DynamoDB
  3. แสดงกราฟและตัวเลขสรุป พร้อมกรองตามช่วงเวลาหรือผู้ใช้
